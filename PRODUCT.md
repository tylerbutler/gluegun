# Product

## Register

brand

## Users

Gluegun serves Gleam and BEAM developers who need a typed HTTP client wrapper around Erlang Gun. They are evaluating whether Gluegun fits production client work, learning the package's connection/request/message model, or returning for exact API and workflow guidance.

## Product Purpose

Gluegun exists to make Gun's HTTP/1.1, HTTP/2, and WebSocket capabilities feel approachable and reliable from Gleam without hiding the underlying streaming model. The website should help developers understand the package quickly, trust its security and error-handling choices, and move from install to working code with minimal friction.

## Brand Personality

Bold, playful, high-energy, and technically trustworthy. The brand can be vivid and memorable, but the documentation voice should stay precise, direct, and useful for developers making production decisions.

## Anti-references

Avoid toy-like candy branding that undercuts technical trust. Do not make the site feel like a novelty wrapper, a generic SaaS documentation template, or a sterile gray/blue developer portal with no point of view.

## Design Principles

- Lead with capability, then prove it with concrete Gleam examples.
- Preserve technical confidence even when the visual system is energetic.
- Make the streaming model explicit instead of smoothing over important tradeoffs.
- Keep adoption paths short: install, connect, request, stream, WebSocket, troubleshoot.
- Let brand personality sharpen comprehension rather than decorate around it.

## Accessibility & Inclusion

Target WCAG AA. Future design work should preserve full keyboard access, readable contrast for body and placeholder text, visible focus states, semantic structure, and reduced-motion fallbacks for any animation.
