## v1.0.0 - 2026-07-17


#### Major Release

##### Initial release

Gluegun is a typed Gleam wrapper for the Erlang Gun HTTP client, covering
connection options, HTTP request streaming, asynchronous message decoding,
high-level HTTP response collection, and WebSocket helpers. Ships with
documentation and examples.


#### Added

##### Error and informational response types

`error.UnsupportedFeature(String)` reports preflight capability checks,
such as rejecting WebSocket upgrades on HTTP/2. `response.Informational`
is a named record (`Informational(status:, headers:)`) for constructing
and pattern-matching `1xx` responses.

##### A focused public API surface

Generated docs cover only public modules (no FFI or test-only helpers).
Connection and Stream handles are exported from their public modules.
Raw WebSocket dynamic option setters live under `gluegun/websocket/raw`.
The root `gluegun` facade covers the minimal common path, including
`await_up`.

##### Consistent naming across the request/response API

Header helpers follow one convention throughout: `add_headers` appends,
`with_headers` replaces. Other names describe what they do:
`websocket.send_close_frame`, `gluegun.new_request`, `request.start_stream`,
and `client.request_options`.

##### TLS configuration builders for connection options

The typed `gluegun/tls` module and `connection.with_tls_opts` let callers
configure peer verification, CA sources, TLS versions, SNI, and client
certificate files, rather than relying on Gun's insecure `verify_none`
default.

##### Expanded documentation

Added a stability/semver policy page and expanded the message-flow guide
to cover connection lifecycle (`await_up`, close vs. shutdown), fin/half-close
semantics, cancellation post-conditions, flow control, the frame model, and
process ownership. The README's streaming example handles informational
responses. Docstrings cover every variant across the connection, message,
request, response, and websocket modules.


#### Changed

##### Better-formatted reference docs

Reference docs render Gleam-formatted code blocks: typed example
variables (a/b/c), current-module qualifiers elided, multi-line
constructors and function params, sum-type blocks, and deprecation
admonitions.
